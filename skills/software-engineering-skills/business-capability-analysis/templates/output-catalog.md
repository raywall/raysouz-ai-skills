# Output Catalog

Generate under `_business-capability-analysis/`:

| File | Purpose |
|---|---|
| `analysis-manifest.json` | scope, repositories, commits, mode, coverage |
| `evidence-register.md` | traceable observations and contradictions |
| `executive-summary.md` | business purpose, central findings, major risks/opportunities |
| `system-structure.md` | repositories, deployables, architecture, constraints |
| `domain-model.md` | vocabulary, entities, value objects, lifecycles, context candidates |
| `capability-map.md` | business capabilities, maturity, ownership, dependencies |
| `business-processes.md` | end-to-end journeys, decisions, outcomes, diagrams |
| `business-rules.md` | validations, calculations, decisions, states, edge/error cases |
| `integrations-and-data.md` | contracts, integrations, authoritative data, consistency |
| `risks-and-opportunities.md` | current risks and evidence-backed opportunities |
| `gaps-and-decisions.md` | unknowns, contradictions, required validation |
| `traceability-matrix.md` | capability/journey/rule/data/system/evidence links |

All Markdown artifacts must state current facts separately from `INFERRED`,
`GAP`, and `PROPOSED` content.
