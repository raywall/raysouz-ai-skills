# Operation 05 - Documentation and Handoff

1. Generate project documentation from `templates/project-documentation.md`.
2. Update the evidence register, current-state model, context map, decomposition
   plan, service specifications, test strategy, and migration decisions.
3. Run `scripts/scan_artifacts.py` against generated documentation and artifacts.
4. Remove or replace sensitive values and request human review.
5. Document remaining gaps, risks, rollback conditions, and the next approved slice.
6. Verify every asserted business rule and completed acceptance criterion remains
   traceable to evidence.
