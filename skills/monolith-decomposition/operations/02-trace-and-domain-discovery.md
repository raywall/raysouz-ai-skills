# Operation 02 - Trace and Domain Discovery

1. Run `scripts/inventory_codebase.py` for a structural inventory.
2. Inspect approved corporate sources and register evidence before using it.
3. Select representative business journeys with the user.
4. Trace each journey end to end using `references/analysis-method.md`.
5. Extract evidenced decisions, calculations, validations, invariants, states, and
   terminal outcomes.
6. Build a current-state domain vocabulary and map conflicts as open gaps.
7. Identify bounded-context candidates without turning them into services yet.
8. Present the current-state model and ask the user to confirm or correct it.

Do not proceed when a blocking trace gap or unresolved domain-term conflict could
change the boundaries.
