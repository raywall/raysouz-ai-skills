# Go Documentation Standards

Act as a Go documentation expert and enforce documentation as a first-class engineering artifact.
Every exported identifier that lacks a godoc-compliant comment is a production defect.
Documentation is the API contract — treat it with the same rigour as the code itself.

---

## Inviolable Rules

1. **Every exported identifier must have a doc comment** — package, type, function, method, constant, variable
2. **Doc comment starts with the name of the thing** — `// OrderService manages...` not `// This manages...`
3. **First sentence is a complete, standalone summary** — godoc extracts this as the index entry
4. **No `@param` / `@return` JavaDoc tags** — Go uses prose; describe params in the function body comment
5. **File header on every `.go` file** — copyright, package purpose, author, date
6. **`//nolint` and `// TODO` comments are not doc comments** — keep them separate

---

## File Header Template

Every `.go` file begins with this header:

```go
// Copyright (c) [YEAR] [Organization Name]. All rights reserved.
// Use of this source code is governed by the [LICENSE] license
// that can be found in the LICENSE file.

// [package-name] implements [one-sentence description of this file's responsibility].
//
// This file is part of the [BoundedContext] bounded context within the [ServiceName] service.
//
// Author:  [team or squad name]
// Created: [YYYY-MM-DD]
// Updated: [YYYY-MM-DD]
```

For generated files:
```go
// Code generated by [tool] DO NOT EDIT.
```

## Package Documentation

Every package must have a doc comment immediately above the `package` declaration.
For packages with multiple files, place the package doc in a dedicated `doc.go` file.

```go
// Package order implements the Order aggregate and its associated domain services
// for the Orders bounded context.
//
// The package contains:
//   - The Order aggregate root and its lifecycle state machine
//   - Value objects: OrderID, OrderStatus, Money, OrderItem
//   - Domain events: OrderPlaced, OrderCancelled, OrderShipped
//   - Repository interface: OrderRepository
//
// Usage:
//
//	order, err := domain.NewOrder(customerID, items)
//	if err != nil {
//	    return fmt.Errorf("new order: %w", err)
//	}
//	if err := order.Place(); err != nil {
//	    return fmt.Errorf("place order: %w", err)
//	}
//
// Thread safety: Order is NOT safe for concurrent use. Callers must synchronise
// access when sharing an Order instance across goroutines (uncommon in practice
// since aggregates are loaded, mutated, and saved within a single transaction).
package order
```

## Type Documentation

```go
// Order is the aggregate root for the order lifecycle in the Orders bounded context.
//
// An Order transitions through the following states:
//
//	Draft → Placed → Confirmed → Shipped → Delivered
//	      ↘ Cancelled (from Draft or Placed only)
//
// State transitions are enforced by domain methods; direct field mutation is prohibited.
// All mutations append to the internal events slice; call DomainEvents to retrieve and
// clear them after persisting the aggregate.
//
// Zero value: Order is invalid in its zero value; always construct via NewOrder.
type Order struct {
    id       OrderID
    status   OrderStatus
    items    []OrderItem
    total    Money
    events   []DomainEvent
    version  int   // optimistic concurrency version
}
```

## Method Documentation

```go
// Place transitions the Order from Draft to Placed state, validating all preconditions.
//
// Preconditions:
//   - Order must be in Draft state; returns ErrInvalidTransition otherwise
//   - Order must have at least one item; returns ErrEmptyOrder otherwise
//   - Total must be greater than zero; returns ErrZeroTotal otherwise
//
// Side effects:
//   - Sets status to StatusPlaced
//   - Appends OrderPlaced domain event to the internal event buffer
//   - Records the placed timestamp
//
// Thread safety: not safe for concurrent calls; callers must synchronise externally.
func (o *Order) Place() error { ... }

// Cancel transitions the Order to the Cancelled state.
//
// Only orders in Draft or Placed status may be cancelled.
// Returns ErrInvalidTransition if the current status does not permit cancellation.
//
// reason is a human-readable string recorded in the CancellationReason field and
// included in the OrderCancelled domain event. It must be non-empty.
func (o *Order) Cancel(reason string) error { ... }

// DomainEvents returns all uncommitted domain events and clears the internal buffer.
//
// This method must be called after successfully persisting the Order aggregate.
// Returned events must be published to the event bus before the caller returns.
// Calling DomainEvents a second time returns an empty slice.
func (o *Order) DomainEvents() []DomainEvent { ... }
```

## Interface Documentation

```go
// OrderRepository defines the persistence contract for the Order aggregate.
//
// Implementations must satisfy the following guarantees:
//   - FindByID returns ErrNotFound if no order exists for the given ID
//   - Save is idempotent when the order version has not changed
//   - Save returns ErrVersionConflict if the stored version differs (optimistic locking)
//   - NextIdentity returns a globally unique, opaque OrderID on every call
//
// Implementations live in the infrastructure layer and must not expose storage-specific
// types (e.g. *sql.Rows, *dynamodb.AttributeValue) in their signatures.
type OrderRepository interface {
    // FindByID retrieves an Order by its unique identifier.
    // Returns ErrNotFound if no order with the given id exists.
    FindByID(ctx context.Context, id OrderID) (*Order, error)

    // Save persists the Order aggregate, creating or updating as necessary.
    // Returns ErrVersionConflict if the order was modified by another writer.
    Save(ctx context.Context, order *Order) error

    // NextIdentity generates a new, unique OrderID.
    // The returned ID is safe to use as a DynamoDB partition key.
    NextIdentity() OrderID
}
```

## Error Type Documentation

```go
// ErrNotFound is returned when an aggregate cannot be located by its identifier.
//
// It is a domain error — callers in the adapter layer should map it to HTTP 404
// or gRPC NotFound status codes.
type ErrNotFound struct {
    // ID is the string representation of the aggregate ID that was not found.
    ID string
}

// Error implements the error interface.
func (e ErrNotFound) Error() string {
    return fmt.Sprintf("not found: %s", e.ID)
}

// ErrInvalidTransition is returned when a state machine transition is not permitted
// from the aggregate's current state.
type ErrInvalidTransition struct {
    // From is the current state of the aggregate.
    From OrderStatus
    // To is the target state that was requested.
    To OrderStatus
}
```

## Constant & Variable Documentation

```go
// OrderStatus represents the lifecycle state of an Order aggregate.
type OrderStatus int

const (
    // StatusDraft is the initial state of a newly created Order.
    // Orders in Draft state have not been submitted and may be freely modified.
    StatusDraft OrderStatus = iota

    // StatusPlaced indicates the Order has been submitted by the customer.
    // No further item modifications are permitted in this state.
    StatusPlaced

    // StatusConfirmed indicates inventory has been reserved for the Order.
    StatusConfirmed

    // StatusShipped indicates the Order has been handed to the carrier.
    StatusShipped

    // StatusDelivered indicates the Order has been received by the customer.
    StatusDelivered

    // StatusCancelled indicates the Order was cancelled before shipment.
    StatusCancelled
)

// DefaultOrderTimeout is the maximum duration allowed for order placement operations.
// Adjust via service configuration; do not modify this constant directly.
const DefaultOrderTimeout = 5 * time.Second
```

## Example Functions (Runnable)

```go
// ExampleOrder_Place demonstrates the order placement flow.
// This example is compiled and run by `go test` to ensure it stays correct.
func ExampleOrder_Place() {
    customer := CustomerID("cust-123")
    items    := []OrderItem{{ProductID: "prod-456", Quantity: 2, Price: NewMoney(1000, "USD")}}

    order, err := NewOrder(customer, items)
    if err != nil {
        log.Fatal(err)
    }

    if err := order.Place(); err != nil {
        log.Fatal(err)
    }

    fmt.Println(order.Status())
    // Output: placed
}
```

## Documentation Quality Checklist

Apply to every PR touching Go code:

- [ ] Every exported type has a doc comment starting with the type name
- [ ] Every exported function/method has a doc comment starting with the function name
- [ ] Every exported interface method has an inline doc comment
- [ ] Every exported constant in a `const` block has an individual comment
- [ ] File header present with copyright, package purpose, author, date
- [ ] Package doc present (in `doc.go` for multi-file packages)
- [ ] Constructors document return value states and error conditions
- [ ] Methods document preconditions, side effects, and thread safety
- [ ] Deprecated identifiers carry `Deprecated:` annotation with replacement
- [ ] Example functions provided for non-obvious APIs
- [ ] `go doc ./...` produces no warnings when run in CI
- [ ] `golangci-lint` with `godot` and `godox` linters enabled and passing

### CI Lint Config (`.golangci.yml` snippet)

```yaml
linters:
  enable:
    - godot      # doc comments must end with a period
    - godox      # flag TODO/FIXME left in exported comments
    - revive     # enforce exported-comment rule
    - staticcheck

linters-settings:
  godot:
    scope: declarations  # only exported declarations
    capital: true        # comments must start with capital letter
  revive:
    rules:
      - name: exported
        arguments: [{ "checkPrivateReceivers": false, "disableStutteringCheck": false }]
```
